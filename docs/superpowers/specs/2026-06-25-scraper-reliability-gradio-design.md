# 爬虫可靠性与 Gradio UI 重构设计

## 目标

在不修改 SQLite 数据库结构和现有 Cookie 管理方式的前提下，提高爬虫的请求可靠性、游标安全性、CSV 写入一致性和可测试性，并删除 Streamlit UI，将 Gradio 作为唯一图形界面。

## 范围

本次修改包括：

- 修复首次请求失败时主循环直接退出的问题。
- 避免持续使用同一个 `nextId` 造成无限循环。
- 为临时网络错误、HTTP 429 和服务端错误增加有限重试与退避。
- 区分可重试错误、响应格式错误和正常结束。
- 使用标准 CSV 写入，避免手动拼接和反复打开文件。
- 每页数据写入后统一提交 SQLite 事务，但不改变数据库表结构或现有覆盖语义。
- 使用原子替换方式更新 `nextId.txt`。
- 删除 Streamlit UI，将 Gradio UI 作为唯一 UI。
- 修复 Gradio 的停止、线程清理、异常展示和重复启动控制。
- 简化 Shell 入口的参数转发。
- 将 `.gitignore` 改为明确的忽略规则，避免新源码被静默隐藏。
- 添加离线自动化测试、依赖声明和静态检查配置。
- 更新中英文 README。

本次明确不包括：

- 修改 `products` 表结构、创建新表或迁移现有数据库。
- 改变商品 ID 主键和 `INSERT OR REPLACE` 的现有语义。
- 修改、取消或重新设计 `cookies.txt` 的 Git 跟踪机制。
- 增加并发抓取、代理池、验证码绕过或其他激进反爬能力。
- 清理当前未跟踪的个人调试脚本和本地数据文件。

## 架构

保留当前的小型 Python 项目形态，不做大规模分层重写。核心职责拆分如下：

- `main.py`：命令行参数、抓取循环、页面结果处理和程序生命周期。
- `tools.py`：HTTP 会话、有限重试、退避、休眠和 checkpoint 原子写入。
- `db.py`：保持现有表结构，提供结构化行写入和每页事务提交。
- `ui_gradio.py`：唯一图形界面，通过可取消的后台任务调用同一抓取循环。
- `tests/`：使用模拟响应验证重试、游标、写入和停止行为，不访问真实 B 站接口。

CLI 和 Gradio 共用同一个抓取控制函数，避免两个入口各自维护循环逻辑。

## 抓取结果模型

单页抓取不再用 `None` 同时表达“结束”和“失败”。内部使用明确状态：

- `SUCCESS`：成功处理页面，并提供新的 `nextId`。
- `FINISHED`：服务端明确返回无下一页，正常结束。
- `STOPPED`：用户请求停止。
- 异常：有限重试耗尽、HTTP 不可重试错误、响应结构不合法或数据处理失败。

主循环只在 `FINISHED` 或 `STOPPED` 时正常退出。异常应显示原因并返回非零 CLI 退出码，Gradio 中则显示错误信息。

## 请求与重试

使用一个复用的 `requests.Session`。单次 HTTP 请求保留连接和读取超时，并遵循以下策略：

- 网络超时、连接错误、HTTP 429 和 HTTP 5xx 可以重试。
- HTTP 429 优先使用合法的 `Retry-After` 秒数。
- 其他重试使用指数退避并添加少量随机抖动。
- HTTP 4xx（429 除外）不重试。
- 重试次数有限；耗尽后抛出带上下文的异常。
- JSON 无法解析、顶层 `code` 缺失或数据字段结构异常视为响应错误，不在主循环中静默吞掉。

现有页面间随机休眠保留，但与错误重试退避分开。

## 游标与停止条件

主循环保存上一次成功处理的游标：

- 若服务端连续返回与当前值相同的非空 `nextId`，立即以“游标无进展”错误停止。
- checkpoint 只在整页数据库和 CSV 写入成功后更新。
- `nextId.txt` 仍只保存游标字符串，不改变用户现有的 `--id` 使用方式。
- 写 checkpoint 时先写同目录临时文件，再使用原子替换，避免中断留下半写文件。
- 正常完成时将 checkpoint 清空。

用户停止采用线程安全的 `threading.Event`。页面请求结束后以及进入下一页前检查停止事件；不强行终止正在执行的网络请求或数据库事务。

## 数据处理

不修改 SQLite schema：

- 保留 `products(id PRIMARY KEY, name, price, market_price, rate, time)`。
- 保留相同商品覆盖旧记录的行为。
- `db.py` 接收结构化字段，不再接收逗号拼接字符串。
- 一页商品使用一个事务；发生异常时回滚该页。

CSV 使用 `csv.writer`：

- 保持现有六列顺序且不增加表头，以兼容已有使用方式。
- 不再删除商品名中的逗号；CSV writer 负责正确引用。
- 每页分别打开一次 total/want 文件并批量写入。
- 商品名中的换行规范为空格，以保持一条商品一条物理记录。
- 缺少必要字段、详情列表为空、价格无法转换或市场价小于等于零时，记录警告并跳过该商品，不导致整页进入无限重试。

## Gradio UI

删除 `ui.py`，README 不再介绍 Streamlit。

Gradio UI 保留现有筛选项，并调整任务控制：

- 启动时创建独立的停止事件和后台任务状态。
- 已有任务运行时拒绝重复启动。
- 停止按钮仅设置停止事件，并提示将在当前请求结束后停止。
- 后台函数始终在 `finally` 中关闭 SQLite 连接。
- 日志以生成器方式持续更新；异常显示清晰消息，界面恢复可再次启动状态。
- CLI 和 UI 使用相同的核心抓取循环，不复制游标和休眠逻辑。

## 项目与依赖

- `main.sh` 改为直接、带引号地转发所有参数给 `main.py`。
- `.gitignore` 改为显式忽略数据库、checkpoint、抓取 CSV、缓存、虚拟环境和系统文件；不更改 `cookies.txt` 当前跟踪状态。
- 添加项目依赖声明，包含 CLI 和 Gradio 运行所需包及测试工具。
- 删除 Streamlit 依赖和文档。
- 中英文 README 同步说明 CLI、Gradio、重试行为和数据输出。

## 测试与验收

测试全部离线执行，不发送真实 HTTP 请求：

- 首次临时失败后能够重试并继续，而不是直接结束。
- 重试耗尽后抛出明确异常，不无限循环。
- HTTP 429 使用 `Retry-After`，5xx 使用退避，普通 4xx 不重试。
- 正常无下一页会结束并清空 checkpoint。
- 相同非空 `nextId` 被识别为无进展。
- 用户停止事件会在页面边界终止循环。
- CSV 能正确保存含逗号的商品名。
- 无效商品数据被跳过，合法商品仍写入。
- 一页写入只提交一次；失败时回滚。
- checkpoint 通过原子替换写入。
- Gradio 后台任务关闭数据库连接并恢复任务状态。

最终验证命令包括：

```sh
pytest
ruff check .
python3 main.py --help
python3 -m compileall main.py tools.py db.py ui_gradio.py
```

验收时不运行真实抓取，以免测试依赖 Cookie、网络状态或远端 API。
